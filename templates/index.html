Two ways to call the same Lambda

Click each button. They invoke the same Lambda function, but through very different paths — one server-proxied through Flask, one direct from the browser. Watch the round-trip times and the invocation_path field in the response.

Stage 1

Server proxy

Browser → Flask /api/hello → SigV4-signed → Lambda Function URL → Lambda

Trust: Flask's EC2 instance role signs the call. User identity is forwarded in the JSON body — Lambda trusts whatever Flask says, because only Flask's role can sign.

Stage 2

Browser direct

Browser → API Gateway → JWT authorizer → Lambda

Trust: Browser fetches a short-lived JWT from /auth/token, presents it to API Gateway. API Gateway validates signature, issuer, audience, and expiry against the gateway's published JWKS. Identity comes from verified claims — Lambda trusts API Gateway's gate.
